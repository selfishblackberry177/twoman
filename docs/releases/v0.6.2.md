# Twoman 0.6.2

Twoman 0.6.2 hardens transport recovery and aligns the released clients with the
latest runtime fixes.

## Highlights

- Added bounded down-stream recovery in the shared transport so stale control/data
  long-polls reconnect instead of hanging indefinitely after broker churn.
- Fixed helper shutdown ordering and cancellation handling so Android and desktop
  helper teardown is faster and less likely to require process-kill fallback.
- Fixed Android transport overflow replay ordering so it matches the shared
  transport behavior.
- Scoped IPv4-preferred DNS filtering explicitly to Android VPN mode instead of
  applying it implicitly to every runtime mode.
- Hardened broker runtime behavior:
  - bind payload crypto to the token that actually authenticated the peer
  - roll back helper/agent stream mappings immediately if agent OPEN queueing fails
- Added focused regression coverage for transport recovery, Android transport
  ordering, broker runtime behavior, helper shutdown, and helper socket cleanup.

## Operational Notes

- Hidden-agent transport recovery hardening is intended to self-heal the stale
  control down-poll failure mode that previously required a manual service
  restart on the hidden server.
- The remaining public-host limitation is Passenger/shared-host runtime churn.
  That is operational, not a client-protocol regression.
