## Twoman 0.6.0

Twoman 0.6.0 is the feature release after 0.5.0. It adds the new desktop app,
expands Windows support, hardens the transport/runtime path, and aligns public
host deployment around the current camouflage and health-check model.

### Added

- New Tauri-based desktop app for Linux and Windows.
- Desktop profile manager, share management, and bundled sidecar packaging.
- Windows tunnel mode in the desktop app, including packaged tunnel sidecars.
- Shared SOCKS and shared HTTP proxy runtime support for desktop clients.
- Managed host deploy flow with randomized Persian-style public routes.
- Branded root camouflage site deployment and stricter public health endpoint handling.

### Improved

- Android client lifecycle handling for proxy and VPN modes.
- Windows tunnel startup and recovery behavior.
- Desktop shutdown behavior so connect and disconnect flows return to a clean state.
- Transport and deployment defaults for the current broker and hidden-agent runtime model.
- Release packaging for Linux and Windows handoff builds.

### Fixed

- Desktop shared HTTP gateway handling.
- Android VPN foreground-service and runtime lifecycle regressions seen on newer Android releases.
- Public-facing 404 behavior so deployed host routes present a consistent camouflage experience.

### Validation

- Android proxy and VPN flows validated on a physical Pixel 6a.
- Desktop runtime validated with:
  - `bash tests/run_desktop_client_e2e.sh`
  - `cargo test live_connect_share_disconnect_flow -- --ignored --nocapture`
- Targeted regression coverage added for helper shutdown, transport ordering, lane boundaries,
  and broker data queue behavior.
