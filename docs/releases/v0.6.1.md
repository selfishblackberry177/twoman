## Twoman 0.6.1

Twoman 0.6.1 is the stabilization release on top of 0.6.0. It focuses on the
live slow/stall reports, especially under Android VPN and mixed real-client load.

### Fixed

- Hidden-agent IPv6 literal origin stalls now fail fast instead of burning the
  full connect timeout on unreliable IPv6 egress paths.
- Helper DNS handling now suppresses AAAA answers when IPv4-preferred VPN mode
  is enabled, reducing bad IPv6 origin opens.
- Small DNS exchanges stay on the control lane so `OPEN`, `DATA`, and `FIN`
  remain ordered during bursty VPN startup traffic.
- Android VPN DNS presentation is back on the standard `VpnService` path using
  real upstream DNS servers, which restores browser compatibility while keeping
  the newer transport-side DNS fixes.

### Improved

- Hidden-server deploy defaults and sample config now expose the IPv4/IPv6
  origin controls directly.
- Release checklist and manual deploy docs now match the current runtime and
  health-check expectations.

### Validation

- Signed Android release validated on a physical Pixel 6a:
  - VPN connect
  - Chrome load of `https://example.com`
  - disconnect back to `Stopped`
- Focused regression coverage:
  - `python3 -m unittest tests/test_helper_dns.py tests/test_dns_lane_selection.py tests/test_hidden_agent_connect.py tests/test_helper_shutdown.py`
- Desktop regression coverage:
  - `bash tests/run_desktop_client_e2e.sh`
