# Twoman 0.7.0

Twoman 0.7.0 is the transport and deployment consolidation release.

This release turns the `0.7` transport refresh into the official shipped
version. The main themes are:

- backend-aware transport profiles
- dedicated DNS handling for tunnel mode
- managed-host Node improvements
- Cloudflare-friendly public-host guidance
- separate hidden-side outbound proxy routing

## Highlights

- Added broker-advertised transport capabilities and profile selection:
  - helpers and agents now follow broker-advertised profiles instead of forcing
    one transport shape across every backend
  - shared hosts stay on conservative bounded HTTP polling profiles
  - managed Node-capable hosts can advertise `managed_host_http` and
    `managed_host_ws`
- Added a dedicated DNS protocol path for tunnel mode:
  - VPN DNS is no longer modeled as ordinary short-lived proxy streams
  - Android VPN reliability improves because DNS is now a first-class protocol
    subsystem
- Added hidden outbound proxy routing:
  - the hidden server can now use one proxy for `server -> public host`
  - and a separate proxy for final `server -> internet` egress
  - production can now route final egress through local WireProxy/WARP when
    needed
- Added managed-host Node broker improvements:
  - local Node HTTP and WebSocket end-to-end coverage now ship in-tree
  - the managed-host transport benchmark harness is now included for direct
    profile comparison
- Added Android profile import improvements:
  - profile sharing and profile import now match the current `0.7` runtime flow

## Cloudflare Notes

Cloudflare is an important operational path in `0.7.0`.

- A Cloudflare-proxied hostname can front the public Twoman host cleanly while
  keeping the public host in-path.
- This can improve public-host reachability when the hidden server reaches
  Cloudflare more reliably than the origin IP directly.
- Cloudflare fronting does not remove the need for hidden-side WARP if the
  hidden server still cannot reliably reach the proxied hostname.
- Cloudflare fronting also does not mean WebSockets should be forced. Twoman
  still follows the broker-advertised profile for the actual backend and route.

## Production Notes

- The current audited shared host still recommends `shared_host_safe`.
- The current managed Node path has the highest upside, but only when the host
  actually advertises that capability.
- Hidden-side WARP remains a reachability and egress tool, not a desktop or
  Android client requirement.
- Camouflage behavior remains in place. This release improves transport and
  deploy behavior without removing the public camouflage model.
